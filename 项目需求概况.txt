请帮我用 Python 的 Streamlit 框架开发一个名为“💰 我的财务健康看板”的个人记账数据处理 Web 应用。

**核心场景**：这是一个面向普通人的日常收支管理工具。用户打开网页后，不需要理解编程，就能看到自己的钱花在了哪里、是否超支。

**必须包含的具体功能**：

1. **内置示例数据（模拟真实生活）**：
   - 自动生成 60 条模拟流水数据，覆盖 2026年6月1日至6月30日。
   - 数据字段包含：日期、类别（餐饮、交通、购物、房租/水电、娱乐、医疗）、收支类型（收入/支出）、金额（元）、备注。
   - 模拟数据要贴合现实：比如早晚餐 15~30元，地铁 5元，周末大餐 200元，月底发工资 8000元，房租 1500元。让用户一运行就有代入感。

2. **数据录入与上传页**：
   - 支持手动添加一笔收支（选择日期、类别、输入金额、备注），并追加到当前数据表中（使用 `st.session_state` 持久化）。
   - 支持上传 CSV 文件（自动识别列名：日期、类别、金额等），覆盖或追加数据。

3. **趋势分析页（核心可视化）**：
   - 使用 Plotly 绘制 **“每日收支变化组合图”**：X轴为日期，柱状图显示每日总支出，折线图显示每日累计余额（收入-支出），左侧Y轴为金额。
   - 使用 Plotly 绘制 **“分类支出占比环形图（饼图）”** ，直观展示钱主要花在哪里。
   - 添加一个时间滑块或月份选择器，支持切换查看不同月份（如果用户有多月数据）。

4. **财务诊断与预警页（亮点功能）**：
   - 顶部显示三个核心指标卡片：**本月总收入**、**本月总支出**、**本月结余**（使用 `st.metric` 展示，结余为负时数字变红）。
   - 自动计算“恩格尔系数”（食品支出/总支出），并给出解读（如：系数>40%提示“饮食开销较大”）。
   - **预警规则**：当单项类别（如餐饮）支出超过总支出的 35% 时，页面自动弹出提醒（`st.warning` 或 Toast 提示）：“本月餐饮占比过高，建议减少外卖频率！”
   - 展示“超支日历”：在日期表格中，将日支出 > 日均支出 * 1.5 的日期标红。

5. **数据导出**：
   - 提供一个“导出当前完整数据为 CSV”的下载按钮。

6. **UI 视觉设计（必须严格参考项目一风格）**：
   - 全局背景使用蓝紫色到粉色的柔和渐变（`#667eea` -> `#764ba2` -> `#f093fb`）。
   - 所有卡片采用毛玻璃效果：`background: rgba(255, 255, 255, 0.12); backdrop-filter: blur(15px); border-radius: 25px; border: 1px solid rgba(255,255,255,0.25);`。
   - 数字和标题要有质感，使用柔和的阴影。
   - 按钮悬停要有缩放动画（`transform: scale(1.03)`）。
   - 所有交互（添加记录、切换月份）都要有流畅的反馈（`st.toast` 提示）。

**技术栈**：`streamlit`, `pandas`, `numpy`, `plotly`, `datetime`。

**额外细节要求**：
- 将“类别映射”和“日期解析”封装成单独函数，方便用户自定义。
- 代码要有详细中文注释，便于我后续修改。

请生成完整可运行的 app.py 单文件或合理拆分模块，并附带 requirements.txt。
